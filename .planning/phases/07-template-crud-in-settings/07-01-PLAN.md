  <files>src/hooks/use-hardware-templates.ts, src/hooks/use-hardware-templates.test.ts</files>
  <behavior>
    - addTemplate(name: string) creates a new template with uuid, empty items array, returns the new id
    - addTemplate with empty/whitespace name is rejected (returns null or throws)
    - addTemplate with duplicate name (case-insensitive) is rejected
    - renameTemplate(id, newName) updates the template name
    - renameTemplate with empty/whitespace or duplicate name is rejected
    - updateTemplateItems(id, items: DoorHardwareEntry[]) replaces the template's items array
    - deleteTemplate(id) removes the template from settings.hardwareTemplates
    - toggleTemplateItem(templateId, hardwareId) adds item with qty 1 if not present, removes if present
    - updateTemplateItemQuantity(templateId, hardwareId, quantity) sets quantity on existing item (min 1, integer)
    - templates getter returns current settings.hardwareTemplates
  </behavior>
  <action>
    Create `src/hooks/use-hardware-templates.ts` with a `useHardwareTemplates` hook that manages template CRUD via `useAppStore().setState`. The hook provides:

    1. `templates` — reads `state.settings.hardwareTemplates`
    2. `addTemplate(name: string): string | null` — validates name is non-empty and unique (case-insensitive trim comparison against existing template names), creates template with `{ id: uuidv4(), name: name.trim(), items: [] }`, returns new ID or null on validation failure
    3. `renameTemplate(id: string, newName: string): boolean` — validates non-empty and unique (excluding self), updates name, returns success
    4. `deleteTemplate(id: string): void` — removes template by id (no referential integrity check needed per CONTEXT.md — templates have no back-reference to line items)
    5. `toggleTemplateItem(templateId: string, hardwareId: string): void` — if hardwareId is in template's items, remove it; if not, add `{ hardwareId, quantity: 1 }`
    6. `updateTemplateItemQuantity(templateId: string, hardwareId: string, quantity: number): void` — set quantity to `Math.max(1, Math.round(quantity))` on the matching DoorHardwareEntry

    All mutations use `setState(prev => ...)` pattern. Use `useCallback` on each method (dependency: `[setState]`). Import `v4 as uuidv4` from `uuid`.

    For the test file, use `// @vitest-environment jsdom` pragma (hook needs React context). Create a test wrapper that renders the hook inside `AppStoreProvider`. Use inline factory helpers for test data. Test all behaviors listed above, including validation edge cases.
  </action>
